# Chat Rules for AI Dev Team in GitLab

## Guidance Preferences

- Prefer repository-specific guidance over generic advice
- Keep suggestions small and demoable
- When proposing code changes, explain why they matter
- When suggesting tests, prioritize changed code paths
- When suggesting security fixes, prefer low-risk remediations

## Output Style

- Use bullet points for clarity
- Be concise and actionable
- Avoid walls of text
- Provide next steps explicitly

## Scope

- Hackathon-scoped prototype thinking
- Small, reviewable pull requests
- Testing that focuses on modified code, not entire coverage
- Security fixes that minimize risk and change surface area
