📊 Common Questions

Q: Where's my data stored?
A: Analytics live in browser localStorage. No server storage (for now).
Q: Can I use this offline?
A: No - needs internet for Claude API calls. Dashboard needs MCP server.
Q: How much does it cost?
A: You pay Claude API usage only (~$0.01-0.10 per agent run).
Q: Can I customize the agents?
A: Yes! Edit backend/agents-config.js and update the system prompts.

🚀 Getting Started (5 Minutes)

Prerequisites

  • Node.js 16+ installed
  • Claude API key from console.anthropic.com
  • 5 minutes of your time

Step 1: Clone & Install

git clone https://gitlab.com/ai-dev-team-labs/ai-dev-team-gitlab cd ai-dev-team-labs npm install

Step 2: Set API Key

echo "CLAUDE_API_KEY=sk-ant-xxxxxxxxxxxxx" > .env

Replace sk-ant-xxxxxxxxxxxxx with your actual API key from Claude console.

Step 3: Start the Product

npm start

Step 4: Open in Browser

  • Landing Page: http://localhost:3002/
  • Dashboard: http://localhost:3002/dashboard
  • MCP Server: http://localhost:3001

✅ You're Done!

Your product is now running. Try selecting an agent and testing it with a request.

🐛 If Something Breaks

Port already in use?
lsof -i :3002 # Check what's using port 3002 kill -9 PID # Kill the process
API key invalid?
Check your .env file and get a new key from console.anthropic.com
Dashboard blank?
Refresh page, check browser console (F12) for errors

💡 User Guide

Understanding Your 4 Agents

👥 Onboarding Copilot

Helps new contributors understand your codebase and get productive quickly.

Use Cases:

  • Onboarding a new team member
  • Creating an onboarding checklist
  • Identifying good starter tasks
  • Explaining repository structure

Example Request:

"I'm a new developer. Help me understand this codebase and what I should work on first."

📋 Feature Planner

Converts requirements into detailed, actionable implementation plans.

Use Cases:

  • Plan a new feature
  • Estimate implementation effort
  • Identify affected areas
  • Create test strategies

Example Request:

"Plan the implementation of dark mode for our application."

✅ Test Generator

Analyzes your code changes and identifies missing tests.

Use Cases:

  • Identify test coverage gaps
  • Suggest test cases for new features
  • Find edge cases
  • Plan test strategy

Example Request:

"I added a login form. What tests am I missing?"

🔒 Security Patch Agent

Reviews security vulnerabilities and proposes safe, tested fixes.

Use Cases:

  • Patch security vulnerabilities
  • Review code for security issues
  • Propose security improvements
  • Validate vulnerability fixes

Example Request:

"I have a CVE-2024-1234 vulnerability in Express.js. How should I patch it?"

🔄 Using Browser Steps (Advanced)

Make agents smarter by letting them research the web:

  1. Click "+ Add Step" in dashboard
  2. Choose action: navigate, click, type, screenshot, extractText, scroll, wait
  3. Fill in parameters
  4. Click "Test Agent"

Example: Feature Planner with design research

  • Step 1: Navigate to https://design.company.com/design-system
  • Step 2: Screenshot
  • Step 3: Extract Text (selector: .patterns)
  • Input: "Plan dark mode implementation"
  • Result: Agent-aware implementation plan!

💡 Tips for Better Results

✅ Be Specific

Good: "Fix the login form validation bug where empty emails are accepted"

✅ Provide Context

Good: "Plan adding dark mode to our React app that uses Tailwind CSS"

✅ Use Browser Steps

Add steps to let agents research your codebase and design systems

✅ Iterate

First request: Get initial plan. Second: "Refine step 3"

🌐 Deployment Guide

Option 1: Netlify (Landing Page - EASIEST)

What you get: Professional marketing site

Cost: Free

  1. Visit app.netlify.com
  2. Click "Import an existing project"
  3. Select "GitLab" → Choose ai-dev-team-labs/ai-dev-team-gitlab
  4. Netlify auto-detects netlify.toml
  5. Click "Deploy Site"
  6. Wait 2-3 minutes
  7. Live at: https://ai-dev-team-labs.netlify.app/

Option 2: Heroku (Full Stack - RECOMMENDED)

What you get: Everything (landing page + dashboard + agents + browser)

Cost: ~$5-7/month

  1. Sign up at heroku.com
  2. Install Heroku CLI
  3. Login: heroku login
  4. Create app: heroku create ai-dev-team-labs
  5. Set API key: heroku config:set CLAUDE_API_KEY=sk-ant-xxxxx
  6. Deploy: git push heroku master
  7. Open: heroku open
  8. Live at: https://ai-dev-team-labs.herokuapp.com/

Option 3: Railway (Modern Alternative)

What you get: Full product like Heroku, easier setup

Cost: Pay-as-you-go (~$5-10/month)

  1. Visit railway.app
  2. Create new project
  3. Connect to GitLab repository
  4. Add CLAUDE_API_KEY environment variable
  5. Deploy automatically
  6. Live at: https://your-project.railway.app/
