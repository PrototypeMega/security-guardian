# AI Dev Team Labs - GitLab Duo Agent Platform

An autonomous AI agent for GitLab that automatically generates onboarding guides and starter issues for new contributors.

## Quick Start (5 minutes)

### 1. Clone and Install
```bash
git clone <repo-url>
cd AI-Dev-Team-Labs
npm install
```

### 2. Configure Environment
```bash
cp .env.example .env
# Edit .env with your GitLab token and Claude API key:
# - GITLAB_API_TOKEN: Your GitLab personal access token
# - CLAUDE_API_KEY: Your Anthropic Claude API key
# - GITLAB_WEBHOOK_SECRET: A random secret for webhook validation
```

### 3. Start the System
```bash
# Terminal 1: Start webhook server
npm run dev

# Terminal 2: Start event orchestrator
npm run cli
```

### 4. Test Webhook
```bash
curl -X POST http://localhost:3000/webhooks/gitlab \
  -H "Content-Type: application/json" \
  -H "X-Gitlab-Token: test-webhook-secret" \
  -d '{"object_kind":"push","project":{"id":1,"name":"test-repo"},"user_username":"test-user"}'
```

Expected response: `202 Accepted` with event ID.

## Architecture

```
GitLab Push Event
       ↓
[Webhook Listener]
  - Validates HMAC signature
  - Queues event in SQLite
       ↓
[Event Orchestrator CLI]
  - Polls database for queued events
  - Routes to appropriate agent
       ↓
[Onboarding Copilot Agent] (Phase 2)
  - Fetches repo structure via GitLab API
  - Analyzes tech stack
  - Calls Claude for reasoning
  - Generates markdown guide + starter issues
  - Posts results as GitLab issue
```

- **Event**: Incoming GitLab events (push, merge_request, issue)
- **AgentRun**: Logs of agent executions
- **OnboardingGuide**: Generated onboarding guides (Phase 2)

## Troubleshooting

### Server won't start
- Check `.env` file exists and `DATABASE_URL` is set
- Ensure port 3000 is not in use: `lsof -i :3000` (macOS/Linux)

### Webhook signature validation fails
- Verify `X-Gitlab-Token` header matches `GITLAB_WEBHOOK_SECRET` in `.env`
- Check that you're POSTing raw JSON (not form data)

### Database is locked
- Stop all processes (server + orchestrator)
- Delete `.db/dev.sqlite*` files (will be recreated on next run)
